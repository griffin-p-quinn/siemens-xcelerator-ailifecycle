# Siemens Capital — Agent Skill

## What It Is
Siemens Capital is the electrical/electronic system (EWIS) design platform — wiring harnesses, schematic capture, wire routing, connector management, and electrical BOM. It is a **desktop application** (Capital Studio) with **no public agent API**. Griffin's **capitalMCP** wraps the Capital REST layer where available.

**Agentic-readiness: 🔴 Custom bridge required — the hardest wall (2/10).**

---

## Integration Layer(s)

| Interface | Form | Reality |
|-----------|------|---------|
| Capital REST API (`capital/rest/v1`) | OpenAPI | **Query / EBOM / write-back only** — NO ECAD authoring over REST |
| **capitalMCP** (Griffin) | TypeScript MCP server | Dynamically wraps the Capital REST API; tools: `list_harnesses`, `get_harness_details`, `get_wire`, `get_connector`, `get_ebom`, `create_wire`/`update_wire` |
| Capital Studio | Desktop GUI | Primary authoring surface — not scriptable for agents |
| Capital Integrator | Multi-domain integration | File/PLM (Teamcenter) handoff |
| Project XML | File | `SensorJunctionBox_project.xml` — Capital project file, generated by Griffin's Project XML generator |

> **Honest caveat:** There is **no public API/CLI/SDK for ECAD authoring**. Real harness *design* stays in Capital Studio. Agents can read/query harness data and write EBOM/wire attributes, but cannot author schematics headlessly.

---

## Deployment & Headless / Server Architecture
- Because ECAD authoring cannot be driven over REST, the agentic pattern is **read/query via capitalMCP + file/PLM handoff for authoring**.
- Griffin's bridge: **Capital Java Bridge + Project XML generator**; harness is materialized in NX (CapitalNXIntegration) rather than authored via API.
- For write-back, the REST v1 EBOM/wire endpoints run behind an HTTP load balancer; no headless *authoring* session exists.
